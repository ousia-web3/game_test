# UI 및 리듬 렌더링 버그 수정 계획

[재미니님], 리듬 게임 사양에 맞춘 UI 정렬 및 렌더링 문제를 해결하기 위한 계획입니다.

## 1. 인트로 화면 UI 중앙 정렬 수정
- **문제 원인**: 인트로 화면의 `.content` 영역 내에서 요소들이 중앙 정렬되지 않아 버튼과 레벨 선택 바가 치우쳐 보였습니다.
- **해결 방안**: `styles.css`의 플렉스 정렬 스타일을 보정하여 모든 설정 UI를 수직/수평 중앙에 배치합니다.

## 2. 리듬 게임 캔버스 렌더링 및 판정선 정렬
- **문제 원인**: 게임 시작 시 캔버스 리사이즈 로직이 즉시 반영되지 않아 판정선(`judgeLineY`) 위치가 어긋나거나 화면이 표시되지 않는 현상이 있었습니다.
- **해결 방안**: `app.js`에서 `startGame` 함수 시작 시 `resize()`를 강제 실행하고, 리듬 비트 처리에 필요한 캔버스 좌표계를 초기화합니다.

## 작업 완료 사항
1. `public/styles.css` 수정 완료
2. `public/app.js` 내 리사이즈 및 초기화 로직 보정 완료
3. 리듬 게임 컨셉에 따른 조작 피드백 UI 추가 확인
