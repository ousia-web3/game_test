# PRD (Product Requirement Document)

## 목표
짧은 시간 안에 플레이 가능한 **리듬 액션 웹 MVP**를 제공하고, 유저가 고유한 8비트 음악과 비트에 맞춰 조작하며 기록을 경쟁할 수 있게 한다.

## 대상 사용자
- 짧고 반복 플레이가 가능한 리듬 게임을 선호하는 사용자
- 8비트 복고풍 사운드와 아케이드 감성을 좋아하는 사용자
- 기록 경쟁과 콤보 달성을 즐기는 사용자

## 사용자 흐름
1. 사용자는 메인 인트로에서 게임의 8비트 테마와 **레인별 조작법**을 확인한다.
2. 생명 코인을 설정하여 초기 생명력을 결정한다.
3. 음악 비트에 맞춰 떨어지는 **계단 노트**를 정확한 타이밍에 입력한다.
4. 판정(Perfect/Great/Good/Miss)에 따라 점수와 생명력이 변동된다.
5. 생명이 0이 되면 결과 화면에서 닉네임과 함께 기록을 등록하고 리더보드를 확인한다.

## 기능 요구사항

### 인트로 및 설정
- 생명 코인 수량 조절(1~5개)을 통해 난이도 및 생명력을 설정할 수 있어야 한다.
- 레벨 선택을 통해 게임의 시작 속도(BPM)를 조절할 수 있어야 한다.
- 시작 전 조작법(`A/S/D` 또는 방향키)과 목표가 명확히 명시되어야 한다.

### 게임플레이 (리듬 엔진)
- **Web Audio API 시퀀서**: 배경음악과 비트 싱크가 정확히 맞아야 한다.
- **3레인 노트 시스템**: 왼쪽, 중앙, 오른쪽에서 떨어지는 노트를 처리해야 한다.
- **판정 시스템**: 판정선과의 거리에 따라 Perfect, Great, Good, Miss를 계산한다.
- **콤보 시스템**: 연속 성공 시 점수 배율이 증가해야 한다.
- **레벨 디자인**: 총 10개의 레벨이 제공되며, 레벨에 따라 고유 음원과 상승된 BPM이 적용된다.

### 결과 및 리더보드
- 최종 점수, 최대 콤보, 달성 거리를 표시해야 한다.
- 닉네임(3자리 영문/숫자) 입력 후 서버(SQLite)에 기록을 등록할 수 있어야 한다.
- 상위 랭킹 정보를 즉시 확인할 수 있어야 한다.

## 비기능 요구사항
- 실시간 리듬 판정을 위해 레이턴시를 최소화해야 한다.
- 별도의 음원 파일 없이 코드로 생성되는 8비트 사운드로 용량을 최적화한다.
- 모바일 가로폭 환경에서도 조작이 가능하도록 대응한다.

## 성공 기준
- 음악 비트와 노트 낙하 타이밍의 일치
- 판정 및 콤보 시스템의 정상 동작
- 점수 등록 및 리더보드 반영 완료
