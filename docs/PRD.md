# PRD

## 목표
짧은 시간 안에 플레이 가능한 웹 MVP를 제공하고, 유저가 인트로에서 게임의 정체성을 느낀 뒤 바로 플레이하고 기록을 남길 수 있게 한다.

## 대상 사용자
- 짧고 반복 플레이가 가능한 웹 아케이드 게임을 원하는 사용자
- 기록 경쟁을 좋아하는 사용자

## 사용자 흐름
1. 사용자는 메인 인트로에서 게임 테마와 조작법을 확인한다.
2. 생명 코인을 주입해 시작 생명력을 설정한다.
3. 게임 중 좌우 이동으로 안전한 계단을 밟고 코인을 수집한다.
4. 생명이 0이 되면 결과 화면으로 이동한다.
5. 닉네임과 함께 점수를 등록하고 리더보드를 본다.

## 기능 요구사항

### 인트로
- 생명 코인 수량을 1~5개 범위에서 조절할 수 있어야 한다.
- 코인 수량에 따라 시작 생명력이 달라져야 한다.
- 시작 전 조작법과 목표가 보여야 한다.

### 게임플레이
- 게임은 시작 시 즉시 진행되어야 한다.
- 플레이어는 좌/우 이동이 가능해야 한다.
- 계단은 안전, 함정, 코인 타입을 포함해야 한다.
- 안전 계단 착지 시 점수가 증가해야 한다.
- 함정 계단 또는 적 충돌 시 생명이 감소해야 한다.
- 코인 획득 시 생명이 소폭 회복되어야 한다.
- 시간 경과에 따라 속도가 증가해야 한다.

### 결과 및 리더보드
- 게임 종료 시 최종 점수, 거리, 최고 속도를 보여야 한다.
- 닉네임 입력 후 점수 등록이 가능해야 한다.
- 최근/상위 기록을 즉시 확인할 수 있어야 한다.

## 비기능 요구사항
- 외부 의존성 없이 로컬에서 빠르게 실행 가능해야 한다.
- 브라우저 새로고침 후에도 점수 데이터가 유지되어야 한다.
- 데스크톱과 모바일 가로폭 모두 대응해야 한다.

## 성공 기준
- `npm start` 후 브라우저에서 바로 플레이 가능
- 점수 등록과 재조회가 동작
- 첫 화면, 플레이, 결과 흐름이 끊기지 않음
